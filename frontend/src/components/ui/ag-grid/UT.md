# AG Grid Excel 功能驗證清單 (UT.md)

本文件列出 **AG Grid 客製化封裝元件** 應符合的 Excel 操作行為驗證點。

## 1. 範圍選取與導覽 (Selection & Navigation)

| 測試項目 | 操作步驟 | 預期結果 |
| :--- | :--- | :--- |
| **滑鼠拖曳多選** | 按住左鍵從 A1 拖曳至 C3 | A1:C3 應顯示淡藍色背景，且周圍有深藍色外框。 |
| **Shift + 點擊選取** | 點擊 A1，按住 Shift 點擊 D5 | 選取範圍應擴展為 A1:D5。 |
| **Ctrl + 點擊多選** | 選取 A1:B2，按住 Ctrl 點擊 D1 | D1 應被加入選取，形成兩個不連續的選取範圍。 |
| **鍵盤擴展選取** | 焦點在 A1，按住 Shift + 方向鍵 | 選取範圍應隨方向鍵移動而擴展或縮減。 |
| **捲動同步** | 選取 A1:B2 後向下捲動 | 選取框 (Selection Overlay) 應緊跟儲存格移動，不位移。 |
| **凍結欄選取** | 在有 Frozen Columns 的情況下選取跨越凍結線的範圍 | 選取框應正確覆蓋凍結區與滾動區，無斷裂。 |

## 2. 編輯與快捷鍵 (Editing & Keys)

| 測試項目 | 操作步驟 | 預期結果 |
| :--- | :--- | :--- |
| **Enter 進入編輯** | 在非編輯狀態按 Enter | 該儲存格應進入編輯模式。 |
| **Enter 提交編輯** | 在編輯模式按 Enter | 提交數值，關閉編輯器，焦點移至同一個格子（或依設定移動）。 |
| **單格 Delete/BS** | 選取 A1 按 Delete 或 Backspace | 進入編輯模式且內容為空。按 **Esc** 應能還原舊值。 |
| **範圍 Delete/BS** | 選取 A1:B2 按 Delete 或 Backspace | 範圍內所有可編輯格內容清空。按 **Ctrl+Z** 應能還原。 |
| **Escape 取消** | 選取範圍後按 Esc | 清除所有選取範圍與剪貼簿狀態（螞蟻行軍）。 |

## 3. 剪貼簿操作 (Clipboard)

| 測試項目 | 操作步驟 | 預期結果 |
| :--- | :--- | :--- |
| **複製 (Ctrl+C)** | 選取 A1:B2 按 Ctrl+C | 範圍周圍出現「螞蟻行軍」動畫，系統剪貼簿應存入 TSV 格式文字。 |
| **單格貼上** | 複製 A1，點擊 D1 按 Ctrl+V | A1 的值被貼入 D1。 |
| **範圍貼上 (1:1)** | 複製 A1:B2，點擊 D1 按 Ctrl+V | A1:B2 的內容完全對應貼入 D1:E2。 |
| **智慧重複填充 (鋪磚)** | 複製 A1:A2，選取 D1:D6 按 Ctrl+V | A1:A2 的內容會重複循環貼入（A1, A2, A1, A2, A1, A2）。 |
| **跨欄位貼上** | 複製 A1:B1，選取 D1:G1 按 Ctrl+V | 橫向重複填充（A1, B1, A1, B1）。 |
| **跳過不可編輯格** | 複製內容貼入包含 `editable: false` 的範圍 | 唯讀格內容應保持不變，其餘格子正常更新。 |
| **資料型別轉換** | 將文字 "123" 貼入數字欄位 | 應自動轉換為 `number` 型別而非字串（若有 current value 或 colDef 指定）。 |

## 4. 填充柄操作 (Fill Handle)

| 測試項目 | 操作步驟 | 預期結果 |
| :--- | :--- | :--- |
| **垂直填充** | 拖曳選取範圍右下角小方塊向下 | 出現虛線預覽框，放開後資料依來源範圍重複填充。 |
| **水平填充** | 拖曳選取範圍右下角小方塊向右 | 同上，橫向擴展填充。 |
| **多欄位同時填充** | 選取 A1:B1 兩格，向下拖曳填充柄 | A 欄與 B 欄同時向下填充各自的來源值。 |
| **填充後選取擴展** | 完成填充後 | 選取範圍應自動擴展至包含「來源 + 填充區」的完整範圍。 |

## 5. 歷史紀錄 (Undo / Redo)

| 測試項目 | 操作步驟 | 預期結果 |
| :--- | :--- | :--- |
| **還原單筆編輯** | 手動修改 A1 後按 Ctrl+Z | A1 還原為舊值。 |
| **還原批次貼上** | 貼上 100 格資料後按 Ctrl+Z | 100 格應同時一次性還原為舊值。 |
| **還原填充操作** | 執行填充柄操作後按 Ctrl+Z | 填充出的所有格子一次性還原。 |
| **重做 (Redo)** | Undo 後按 Ctrl+Y 或 Ctrl+Shift+Z | 重新套用剛才被還原的修改。 |
| **堆疊上限** | 連續操作超過 100 步 | 應保留最近的 100 步，最舊的紀錄自動捨棄。 |

## 6. SearchableSelectEditor (自訂下拉選單)

| 測試項目 | 操作步驟 | 預期結果 |
| :--- | :--- | :--- |
| **開啟並搜尋** | 進入編輯，輸入關鍵字 | 選項列表應即時過濾。 |
| **鍵盤導覽** | 使用方向鍵上下移動 | 高亮 (Highlight) 項目隨之移動。 |
| **確認選取** | 按 Enter 或 Tab | 選中高亮項並關閉編輯器。Tab 應額外移動焦點至下一格。 |
| **取消選取** | 按 Escape | 關閉編輯器，不變更原始值。 |
| **滑鼠選取** | 直接點擊列表中的項目 | 完成選取並關閉，焦點回到該格。 |

## 7. 視覺與主題 (Visual & UI)

| 測試項目 | 操作步驟 | 預期結果 |
| :--- | :--- | :--- |
| **深色模式切換** | 切換系統或 App 為 Dark Mode | 背景色、文字、邊框、選取色應自動調整且符合 Notion 審美。 |

## 已修復問題 (Fixed Issues)

在驗證過程中發現並已修復的項目：

- [x] **Shift+Click 範圍選取失效**：修復了 `onCellFocused` 在 `mouseup` 後過早觸發導致選取範圍被重置為單格的問題。現在使用 `nextTick` 確保選取狀態穩定。
- [x] **JS Runtime Error (Undefined 'api')**：修復了 `handleMouseUp` 與 `onCellMouseDown` 中 `api` 變數未定義的錯誤，全面改用 `gridApi.value` 或 `params.api`。
- [x] **清理選取函式缺失**：修復了 `clearSelections` 呼叫了不存在的 `updateRangeSelection` 的問題，已改為正確的重繪邏輯。
- [x] **Redo 快捷鍵支援**：確認並修復了 `Ctrl+Y` 與 `Ctrl+Shift+Z` 的觸發邏輯。
- [x] **雙擊無法進入編輯模式**：由於 `onCellMouseDown` 觸發 `refreshCells` 會重建 DOM 導致雙擊失效。已優化邏輯，若點擊已聚焦格子則延遲或跳過重繪以保留雙擊事件。
- [x] **框選框遮擋凍結欄位**：將 Overlay 透過 `Teleport` 移入 AG Grid 內部的 `.ag-root` 容器，使其遵循原生的層級順序，滾動時會被凍結欄位正確遮擋。
- [x] **大量資料複製效能**：優化了 TSV 字串構建邏輯，使用預配置陣列提高大量儲存格複製時的反應速度。
- [x] **部分欄位無法複製 (valueGetter/格式化問題)**：
    - **問題**：原本手動讀取 `field` 導致 `valueGetter` 運算的欄位（如價格、計算欄位）返回空值，且 `valueFormatter` 呼叫方式不正確導致複製崩潰。
    - **解決方式**：全面改用 `api.getCellValue({ rowNode, colKey })` 獲取原始值，並手動安全調用 `colDef.valueFormatter` 函式來產生最終複製文字。
