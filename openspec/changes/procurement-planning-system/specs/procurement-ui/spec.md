## ADDED Requirements

### Requirement: 採購計畫列表頁
系統前端 SHALL 提供採購計畫列表頁面，顯示所有計畫的摘要資訊，並支援新增計畫。

#### Scenario: 顯示計畫列表
- **WHEN** 使用者進入採購規劃列表頁面
- **THEN** 頁面顯示所有採購計畫的卡片或表格，每筆包含名稱、預計採購日期、狀態、項目數量、預估總金額

#### Scenario: 列表為空
- **WHEN** 系統中沒有任何採購計畫
- **THEN** 頁面顯示空狀態提示，引導使用者建立第一筆計畫

#### Scenario: 新增計畫
- **WHEN** 使用者在列表頁點擊「新增計畫」並填入名稱與預計採購日期
- **THEN** 系統建立計畫，列表自動更新顯示新計畫

#### Scenario: 刪除計畫
- **WHEN** 使用者在列表頁點擊某計畫的刪除按鈕並確認
- **THEN** 系統刪除該計畫，列表自動更新

### Requirement: 採購計畫詳情頁
系統前端 SHALL 提供計畫詳情頁面，顯示計畫基本資訊與所有計畫項目，支援項目 CRUD 操作。

#### Scenario: 顯示計畫詳情與項目列表
- **WHEN** 使用者從列表頁點擊某計畫進入詳情頁
- **THEN** 頁面顯示計畫名稱、預計採購日期、狀態，以及所有項目的表格（設備名稱、規格、數量、預估單價、預估小計、備註）

#### Scenario: 在詳情頁新增項目
- **WHEN** 使用者在計畫詳情頁點擊「新增項目」並填入設備資料
- **THEN** 系統建立項目，項目列表自動更新，預估總金額重新計算

#### Scenario: 在詳情頁編輯項目
- **WHEN** 使用者點擊某項目的編輯功能並修改資料
- **THEN** 系統更新項目，表格自動刷新

#### Scenario: 在詳情頁刪除項目
- **WHEN** 使用者點擊某項目的刪除按鈕並確認
- **THEN** 系統移除該項目，表格與總金額自動更新

#### Scenario: 提交計畫
- **WHEN** 使用者在詳情頁點擊「提交計畫」按鈕
- **THEN** 計畫狀態變為 SUBMITTED，頁面上的編輯與刪除功能變為不可用

#### Scenario: 已提交計畫的唯讀模式
- **WHEN** 使用者檢視 SUBMITTED 狀態的計畫詳情
- **THEN** 頁面以唯讀模式呈現，隱藏新增、編輯、刪除、提交等操作按鈕

### Requirement: 側邊欄導航整合
系統前端 SHALL 在側邊欄加入「採購規劃」導航入口。

#### Scenario: 導航至採購規劃
- **WHEN** 使用者點擊側邊欄的「採購規劃」連結
- **THEN** 系統導航至採購計畫列表頁

### Requirement: 載入與錯誤狀態
系統前端 SHALL 在資料載入與操作失敗時提供適當的 UI 反饋。

#### Scenario: 資料載入中
- **WHEN** 頁面正在從 API 載入資料
- **THEN** 頁面顯示 skeleton loading 效果

#### Scenario: API 操作失敗
- **WHEN** 任何 API 操作發生錯誤
- **THEN** 頁面顯示 toast 通知告知使用者錯誤訊息

#### Scenario: 操作成功通知
- **WHEN** 使用者成功建立、更新、刪除或提交計畫/項目
- **THEN** 頁面顯示 toast 通知確認操作成功
